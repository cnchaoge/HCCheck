# HCCheck — 流程清单

## 整体流程

```
启动 HCCheck.exe
  ↓
① 打开浏览器 → 登录页面
  ↓
② 登录（账号密码自动填入 / 手动登录）
  ↓
③ 导航树自动检测 → 切换到工作台
  ↓
╔══════════════════════════════════════════╗
║           主循环 (while True)            ║
╠══════════════════════════════════════════╣
║ ① 检查工作台是否有未完成任务（续跑）      ║
║    ├ 有 → 读当前节点 → 从对应步骤开始    ║
║    └ 无 → 去普货审验列表取车牌 → 创建    ║
║ ② 车牌含"挂"? → 带挂流程 / 不带挂流程    ║
║ ③ 按节点循环处理各弹窗                    ║
║ ④ 完成 → 回工作台 → 下一辆               ║
╚══════════════════════════════════════════╝
  ↓
列表为空 → 流程结束
  ↓
黑名单持久化（下次启动依然生效）→ 写入 .skip_plates.json
```

|------|------|------|

## 续跑逻辑

工作台读到当前节点后直接映射到步骤常量，跳过前面所有已完成步骤：

| 工作台节点 | 从哪步开始 |
|-----------|-----------|
| 车辆检测 | popup1 |
| 技术岗位审核 | popup2 |
| 业务岗位审核 | popup3 |
| 车辆年审 | popup4 |
| 归档 | popup5 |

### 特殊情况：工作台未找到该车

当 Phase1 点击“创建任务(R)”后页面跳转未完成，主循环读工作台时**还没看到该车**。这种情况：
1. Phase1 返回"工作台未找到 X，可能已完成"，跳出本轮
2. 下一轮主循环重新扫描工作台，**自动续跑**
3. 例：冀J75E8挂（2026-06-24 12:30）就是这样被自动救回的
