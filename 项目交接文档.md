# HCCheck — 运管站货车审验自动化 · 项目交接文档

---

## 🎯 项目背景

- **客户**: 运管站（To G 内部工具）
- **环境**: 运管站 Windows 电脑
- **URL**: `https://221.195.18.1:8181/yg/loginAction.do`
- **系统**: Java Struts 老系统，frameset 结构
- **技术栈**: Python 3.12 + Playwright + Tkinter
- **目标**: 自动跑货车年审流程（批量审验）
- **工具名**: **HCCheck** v1.1

---


## 🔄 完整流程图

```
启动 HCCheck.exe
  ↓
① 打开 Chrome → 登录页面
  ↓
② 自动填入账号密码 → 点登录（或手动登录后继续）
  ↓
③ 导航树自动检测 → 切换到工作台
  ↓
╔══════════════════════════════════════════════╗
║            主循环 (while True)               ║
╠══════════════════════════════════════════════╣
║  ① 切到工作台                                ║
║     ├ 有任务 → 读当前节点 → 从对应步骤续跑   ║
║     └ 无任务 → 去列表取车牌 → 创建新任务     ║
║  ② 车牌含"挂"? → 带挂 / 不带挂 分流          ║
║  ③ 按工作台节点循环驱动各 popup              ║
║  ④ 完成 → 回工作台 → 下一辆                  ║
╚══════════════════════════════════════════════╝
  ↓
列表为空 → 流程结束 ✅
```

## 🧩 帧结构 (Frame)

```

    iframe[name='submitDiag'] — 选择下一节点处理人弹窗

    iframe[name='_Iframe_content'] — 部分内容(含打印链接)
  iframe[name='submitDiagcl'] — 子对话框(选车)
    frame[name='FindFrame_kf'] — 搜索框
    frame[name='main_kef'] — 搜索结果
```

## 🔑 关键配置 (config.py)

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `DEBUG` | False | 调试模式（每步暂停确认） |
| `SINGLE_RUN` | False | 单次模式（跑完一辆停） |
| `MAX_FAIL` | 3 | 失败超限进黑名单 |
| `MAX_CARS` | 0 | 最多跑几辆（0=无限） |
| `HEADLESS` | False | 无头模式（后台运行） |
| `SLOW` | 0.5 | 操作间隔秒数 |
| `LOGIN_USERNAME` | "" | 登录账号 |
| `LOGIN_PASSWORD` | "" | 登录密码 |
| `LOGIN_AUTO_SUBMIT` | True | 自动点登录按钮 |
| `SCHEDULE` | "" | Cron 定时表达式 |

> 日常运行只需在 GUI 设置页配置，无需手动改 config.py

## ⚙️ 核心函数

| 函数 | 功能 |
|------|------|
| `wait_for_login_and_navigate()` | 登录 + 导航树就绪（支持自动填账号密码） |
| `_phase1_select_and_create()` | 选车 + 创建任务（支持续跑） |
| `_check_workbench_for_task()` | 检查工作台已有任务 |
| `read_workbench_node()` | 读工作台当前节点 |
| `open_task_popup()` | 打开任务弹窗 |
| `dispatch()` | 带挂/不带挂分流 |
| `process_unmarked()` | 不带挂流程（5个popup） |
| `process_marked()` | 带挂流程（5个popup） |
| `get_next_plate_from_list()` | 年审列表取第一个无流水号车牌 |
| `_close_all_popups()` | 清理残留弹窗 |
| `wait_until(condition, ...)` | 智能等待条件满足（替代猜时间 pa()） |
| `wait_until_not(condition, ...)` | 智能等待条件消失 |
| `_any_year_check_visible(popup)` | popup 任意 frame 是否有“年度审验”链接可见（wait_until 轮询条件用） |

## 🔧 已知问题

### ✅ 已修复

| 问题 | 方案 |
|------|------|
| 打印链接点不到 | 4种策略暴力查找 |
| 弹窗自动跳转失败 | frame_locator 兜底 |
| 选车牌取最后一个 | 改为 FIFO 第一个无流水号 |
| 失败重试死循环 | SKIP_PLATES 黑名单 |
| 弹窗泄漏 | 异常时 _close_all_popups() |
| 裸 except 吞错误 | 核心路径加日志 |
| 续跑从错误步骤开始 | 读工作台当前节点直接映射 |
| 导航树 ID 写死 | 4种 selector 备选 |
| 归档 confirm 报错 | 全局 dialog handler 自动 accept |
| 黑名单丢失（GUI 重启后不在） | 黑名单持久化到 `.skip_plates.json`（423ec5f） |
| 黑名单藏在设置页不好找 | 独立 `黑名单` Tab + 列表/移除/刷新/清空（ab546fa + 024178f） |
| 服务端“不能创建流程”永久错 | 客户端检测 → 自动加黑名单 → 清理弹窗（c93197c + 1b1aa38 + c10a8fc + 724339e） |
| popup4 提交后工作台假更新 | 提交后 pa(2) → pa(5)（5f73b1d） |
| popup3 不带挂流程双 header | 清理重复 print，仅留 push_status（d65a0fb） |
| popup2 打印预览关不掉 | CDP `Page.setPrintDialogBehavior` 在 main_page 抑制（d65a0fb）→ Lodop.SET_PRINT_MODE 尝试（c10c84d）→ 最终接受 best effort 不一定关（c3c3200） |
| popup4 年度审验链接偶发找不到 | 先点车牌链接让链接显示 + wait_for(state="visible") + input 异常处理（d5478b3） |
| 登录后首次切工作台偶发崩 | wait_for_login_and_navigate 加 3 次重试 + 中间 pa(3) 缓冲（c3c3200） |
| 同车被处理 2 次 | Phase1 点击“创建任务(R)”后多 pa(3) 让后台推工作台（70ccd28） |
| `pa(3)` 猜时间不稳 | 智能等待 `wait_until` / `wait_until_not`（ea3851c） |
| popup4 retry 之间猜时间 | 改 `wait_until(_any_year_check_visible, timeout=3, poll=0.5)` 智能等待（f2d6c2f） |

### ⚠️ 待优化（健壮性优先级）

| 问题 | 优先级 | 说明 |
|------|--------|------|
| popup4 retry 后 `submitDiag` 弹窗残留 → popup3 错乱 | **P0** | 12:30 跑发现：retry 多开弹窗 → dialog 处理旧弹窗 → 节点不变 → 错进 popup3 → 全找不到元素 → 失败 1/3。修复方向：retry 之间先 `wait_until_not(弹窗存在)` + 强制关；popup4 提交后状态断言 |
| 异常分类（RecoverableError/PermanentError/UserInterveneError） | P1 | 所有异常都走 `except Exception` 太粗糙，应分类后上层分支处理 |
| 启动清理（杀遗留 chromium + 清临时文件 + 状态断言） | P2 | 防止上次崩溃污染本次运行 |
| 定时任务调度逻辑（界面配了但未实现） | P2 | GUI 有 cron 输入但实际调度未接 |
| 无头模式批量运行 | P3 | HEADLESS=True 但未充分验证（看不到弹窗） |
